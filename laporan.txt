TUGAS 6 RESTFUL API (LARAVEL) - Middleware

1. Pemahaman dan Pembuatan Middleware
Membuat middleware bernama `Admin` menggunakan perintah artisan yang bertugas sebagai penengah (interceptor) untuk mengecek hak akses/role dari pengguna HTTP Request. Middleware ini memastikan bahwa pengguna yang sedang login (`$request->user()`) memiliki role bertitel `admin`. Jika bukan, akses ditolak dan disajikan JSON berisi pesan "Unauthorized or Forbidden" (403).

2. Pengaturan Routing API (routes/api.php)
Mengatur routing terkait Author, Genre, dan Book menjadi dua buah kelompok hak akses kontrol menggunakan `apiResource()`:
- Rute Publik: Method/Aksi `index` (Read All) dan `show` dibebaskan untuk publik sesuai kriteria tugas menggunakan `.only(['index', 'show'])`. Semua pengunjung tanpa login tetap bisa melihat data.
- Rute Admin: Method/Aksi `store` (Create), `update`, dan `destroy` dilindungi di bawah grup middleware pendeteksi autentikasi `['auth:sanctum', Admin::class]`. Hanya admin yang terkualifikasi dan ber-token sah yang dapat mengeksekusi manipulasi data.

3. Konfigurasi Autentikasi dan Token Sanctum
Membuat `AuthController` untuk menangani *endpoint* login (`POST /api/login`) sehingga sistem dapat menerbitkan token JWT bagi user berdasarkan kredensial (email & password) yang dicocokkan dengan data pada tabel `users`. Model `User` juga sudah dilengkapi dan didukung trait `HasApiTokens` dari Laravel Sanctum sehingga token `Bearer` siap digunakan.

4. Testing Aplikasi Menggunakan POSTMAN
Melakukan berbagai testing operasional simulasi menggunakan aplikasi klien POSTMAN:
- Menguji akses Read All & Show tanpa token: Memastikan *response* data mengembalikan daftar *Author*, *Genre*, dan *Book* sukses.
- Mensimulasikan pengguna sebagai Customer & Admin via `POST /api/login`: Mendapatkan Token Akses masing-masing pihak.
- Menguji API Create, Update, dan Destroy tanpa login atau berbekal Token milik *Customer*: Memastikan sistem menargetkan proteksi error *Blocked/403/Forbidden*.
- Menguji API Create, Update, dan Destroy dengan Token milik *Admin*: Uji coba berhasil menembus autentikasi serta *middleware* untuk menambahkan (via form JSON), mengubah, atau menghapus entitas data.
