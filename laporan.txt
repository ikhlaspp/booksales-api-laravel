TUGAS 4 RESTFUL API (LARAVEL)

1. Fitur Read All Data (Tabel Genre dan Author)
Fungsi `index()` pada `GenreController` dan `AuthorController` mengambil seluruh data dari database dan mengembalikannya dalam format JSON menggunakan `response()->json()`. Endpoint GET `/api/genres` dan `/api/authors` digunakan untuk memanggil fungsi ini.

2. Fitur Create Data (Tabel Genre, Author, dan Book)
Menambahkan method `store(Request $request)` pada masing-masing controller (`GenreController`, `AuthorController`, dan `BookController`). 
Fungsi ini membaca input data, melakukan pengecekan validasi data (seperti validasi required, max, dan numeric), kemudian data tersebut disimpan ke dalam database dengan menggunakan fungsi Eloquent `create()`. Setelah berhasil disimpan, API mengembalikan respons JSON beserta status HTTP 201 Created.

3. Mendaftarkan API Route POST (routes/api.php)
Mendaftarkan endpoints POST baru di dalam `routes/api.php` yang mengarah pada method `store()`:
Route::post('/genres', [GenreController::class, 'store']);
Route::post('/authors', [AuthorController::class, 'store']);
Route::post('/books', [BookController::class, 'store']);

4. Testing API Menggunakan POSTMAN
Melakukan pengujian fitur RESTful API menggunakan aplikasi klien POSTMAN:
- Request GET ke `/api/genres` dan `/api/authors` untuk mencoba fitur Read All data.
- Request POST ke `/api/genres`, `/api/authors`, dan `/api/books` untuk fitur Create. Pengujian ini menggunakan format RAW JSON pada bagian tab 'Body' dan menambahkan `Accept: application/json` pada tab 'Headers' untuk memastikan response tetap berbentuk JSON.
